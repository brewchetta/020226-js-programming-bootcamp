Variables

A variable:

  • ...is a container that stores a value.
  • ...must be declared and assigned a value
  • ...is a dynamic placeholder

Declaring variables:

let changeable = "I am changeable"
const unchangeable = "I am unchangeable"

You may reassign variables declared with let:

changeable = "I am a new value"

You may not reassign variables declared with const. This means anytime you see the = operator or some special operators you cannot use these with a const:

const someNumber = 2
// all three of the following won't work:
someNumber = someNumber + 1
someNumber += 1
someNumber++

Strings

Strings are:

  • ...characters wrapped in quotes
  • ...for storing text
const newString = "I am a string 123 !@#$%^&*"
const emptyString = ""
const notEmptyString = " "
const singleQuoteString = 'I am using single quotation'
const backtickString = `I am declared using backticks`

String interpolation allows us to plug values into the string but only works with backticks:

const interpolatedThing1 = "Hello"
const interpolatedThing2 = "World"

const usingStringInterpolation = `My interpolated string: ${interpolatedThing1} ${interpolatedThing2}`

Concatenated strings are strings that have been "added" together with a +

const concatenatedString = "My concatenated string: " + interpolatedThing1 + " " + interpolatedThing2

A few string methods...

To uppercase and to lowercase:

const uppercaseMe = "make me uppercase"
const uppercase = uppercaseMe.toUpperCase()
// "MAKE ME UPPERCASE"

const lowercaseMe = "MAKE ME LOWERCASE"
const lowercased = lowercaseMe.toLowerCase()
// "make me lowercase"

The .split() method will chop a string into a an array:

const splitMe = "split me into an array"
const split = splitMe.split(" ")
// [ "split", "me", "into", "an", "array" ]

The .join() method will push an array back together into a string:

split.join(" ")
// "split me into an array"

The .slice() method will return a smaller part of the string:

const sliceMe = "slice me up"
sliceMe.slice(6, 8) // "me"
// allows us to grab a smaller part of the string based on a starting and ending point (the 7th letter to the 9th letter)

Numbers

Te modulo operator % returns the remainder of a division operation

5 % 4 // >> 1
5 % 3 // >> 2
5 % 2 // >> 1

If something is cleanly divisible the remainder is 0 which is helpful for determining even numbers:

6 % 2 // >> 0
7 % 2 // >> 1

Math can be used in order to round up, round down, or get a random number between 0 and 1:

Math.ceil(1.1) // 2
Math.floor(1.1) // 1

Math.random()
// 0.5051910743347165

Math.random()
// 0.04570057058789023

Math.random()
// 0.45876204163942524

These can be combined to get a random index for an array:

function getRandomIndex( array ) {
     return Math.floor( Math.random() * array.length )
}

Functions

Functions are:

  • ...a group of reusable code that executes a desired task at a specific time
  • ...a standard operation that can be applied to different variables

The function below has been declared with the name doSomething and accepts three arguments:

function doSomething(thing1, thing2, thing3) {
     return `${thing1} and ${thing2} and ${thing3}`
}

A parameter is the variable name, for example thing1 and thing2 and thing3. You use the name parameter when you are declaring a function.

An argument is the value you pass into the function when it's called. Below the arguments are "I am thing 1", "I am " + "thing 2", and thing3:

const thing3 = "I am thing 3"

doSomething("I am thing 1", "I am " + "thing 2", thing3)

Parameters are a way to give information to the function. The return is a way to take information out of the function.

function giveMeSomething() {
     return "here ya go"
}

const whatIGot = giveMeSomething()
// "here ya go"

With multiple return statements only the first return in the function will work and the others will be ignored:

function multipleReturns() {
     return "first return"
     return "second return"
     return "third return"
}

multipleReturns() // "first return"

Arrow functions are another way of declaring functions:

const myArrowFn = (params) => { "the body of the arrow function" }

// function = (maybe with a parameter) =such=that=> { the body }

Arrow functions with only one line and no curly brackets {} have an "implicit return" which means they automatically return the result of the function:

const automaticallyReturn = () => "I get returned automatically or 'implicitly'"

automaticallyReturn() // "I get returned automatically or 'implicitly'"

Arrow functions can have very little syntax when they have only one parameter and an implicit return.

param => "the body of the arrow function"

JS Objects

Objects are data structures - they hold other pieces of information inside of them:

const someObject = {
     key: "value"
}

Data can be accessed and manipulated in an object:

const person = {
     name: "Chett",
     age: 21
}

person.name // "Chett"
person.age // 21
person.favoriteFood // undefined

person.name = "Chort"
person.age = 100
person.favoriteFood = "pizza"

{
     name: "Chort",
     age: 100,
     favoriteFood: "pizza"
}

The above is using dot notation but bracket notation [] is also a way to access and change information:

person["name"] // "Chort"
person["favoriteFood"] // "pizza"
person["favoriteFood"] = "hawaiian style pizza"

{
     name: "Chort",
     age: 100,
     favoriteFood: "hawaiian style pizza"
}

Bracket notation can be used to dynamically access key/value pairs like using an index:

const statePopulation = {
     alaska: 738737,
     hawaii: 1432820
}

const theStateToGet = "alaska"
statePopulation[ theStateToGet ] // 738737
